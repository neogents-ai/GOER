Building a 16-Role AI Security Team in Claude Code

Stack: Claude Code, Claude Skills (.claude/skills/), NEO Gents Infra  |  Domain: ai_infra / security

What This Is

Claude Code supports custom skills — persistent persona files that give Claude a specialized role, domain knowledge, and output format. This playbook documents how to build a full 16-role security operations team as Claude skills, each scoped to a specific security function (cloud, network, identity, offensive, compliance, etc.) and tailored to a specific infrastructure stack. The result is a fully orchestrated AI security team you can invoke with a single slash command from any terminal session.

Prerequisites

  • Claude Code installed and running (claude CLI)
  • A ~/.claude/skills/ directory (create if it doesn't exist)
  • Knowledge of your target stack — in this build: OCI, WireGuard, Caddy, n8n, SFS-OS

Steps

  1. Create the skills directory structure 
    mkdir -p ~/.claude/skills/ciso
mkdir -p ~/.claude/skills/grc-analyst
mkdir -p ~/.claude/skills/pen-tester
# repeat for each role

    Each skill lives in its own folder. Claude Code looks here when you type /skill-name in any session.

  2. Understand the SKILL.md format

    Each skill is a single SKILL.md file with a frontmatter header and a role definition body:

    ---
name: role-name
description: One-line description — this shows in /help and triggers auto-invoke
---

# Role Name
You are [role]. You do [function]...

## Your domain
...

## Output format
...

    The description field is critical — Claude uses it to decide when to auto-invoke the skill based on context.

  3. Define the orchestrator first (CISO)

    Build the CISO skill as the top-level orchestrator. It knows all 16 roles and can invoke them in sequence for a full security audit.

    # ~/.claude/skills/ciso/SKILL.md
---
name: ciso
description: Chief Information Security Officer. Orchestrates the full security team...
---

# CISO
You are the CISO for NEO Gents. You run full security posture reviews by
delegating to your team. When invoked with /ciso, you call each relevant
role and synthesize findings into an executive summary...
  4. Build skills by color team (security operations model)

    Organize skills using the standard security color wheel — this mirrors how real SOC teams operate:

    ColorSkillsFunction
    Whiteciso, grc-analystGovernance & orchestration
    Redpen-testerOffensive — adversary emulation
    Purplepurple-teamRed→blue bridge, detection validation
    Bluesoc-analyst, incident-response, help-deskDefensive monitoring & response
    Yellowcloud-engineer, devops-engineer, sysadmin, dbaInfrastructure & operations
    Greensecurity-engineer, iam-engineer, network-engineerDetection tooling & access control
    Orangeappsec-engineer, ai-securityApplication & AI security
  5. Wire each skill to your specific stack

    Generic skills are less useful. Each skill should include a section about the actual environment it's protecting:

    ## Primary environment
- Cloud: Oracle Cloud Infrastructure (OCI)
- VPN: WireGuard (site-to-site + peer access)
- Reverse proxy: Caddy
- Automation: n8n
- OS: SFS-OS (custom hardened Linux)

## OCI security checklist
| Check | Pass / Risk |
|---|---|
| Static IP reserved | |
| SSH key-based auth only | |
| Firewall: 22/80/443 only | |
...
  6. Define a consistent output format per skill

    Standardize how each skill reports findings. Example for infrastructure roles:

    [CLOUD] FINDING: <title>
Severity: <critical|high|medium|low|info>
Resource: <resource type and ID>
Detail: <what is wrong and the risk>
Remediation: <specific fix — CLI command or config change>

    Consistent format means you can pipe findings into a database (like db.py) for tracking.

  7. Verify skills are loaded

    Open any Claude Code session and type:

    /help

    All 16 skills should appear in the available skills list. Test each one individually before running the full CISO audit.

    /cloud-engineer   # test a yellow role
/pen-tester       # test the red role
/ciso             # run the full orchestrated audit
  8. Run a full security audit

    With all 16 skills loaded, a single command triggers the full team:

    /ciso

    CISO delegates to each relevant role, collects findings, and returns an executive summary with severity-ranked action items.

Verification

  • Run /help in any Claude Code session — all 16 skills appear in the roster
  • Run /ciso — it invokes sub-roles and returns a structured findings report
  • Run an individual skill like /pen-tester — it responds in its defined output format
  • Check ~/.claude/skills/ — 16 folders each with a SKILL.md file

Why This Matters for Cloud Engineering

  • Demonstrates applied knowledge of security operations models (color wheel, SOC structure) and how they map to real infrastructure roles — directly relevant to cloud security engineer and DevSecOps positions.
  • Shows ability to build tooling around an AI platform (Claude Code skills API) to automate security review workflows — a practical example of AI integration into an engineering operations stack.
  • Each skill is scoped to a specific production stack (OCI, WireGuard, Caddy, n8n) — proving hands-on familiarity with enterprise cloud infrastructure, not just theoretical knowledge.
